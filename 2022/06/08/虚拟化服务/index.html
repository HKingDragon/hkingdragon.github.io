1.虚拟化架构

容器模式

虚拟机运行在传统的操作系统上，创建一个独立的虚拟化实例，指向底层托管操作系统，被称为“操作系统虚拟化”。

主机模式

虚拟机被当作一个应用程序运行在传统的操作系统上，虚拟化的应用程序提供一个全仿真的硬件实例。这种虚拟化架构也被称为“托管” Hypervisor。

裸机模式

虚拟机直接运行在硬件上，有虚拟化软件提供全仿真的硬件环境。这种类型也叫作“裸金属”。

虚拟化监视器（VMM）：主要负责创建、管理和删除虚拟化硬件

半虚拟化，需要修改软件，以便让它知道它运行在虚拟化环境中。对于特定的Hypervisor兼容，包括下面一种或两种情形。

  • 内核半虚拟化（Kernel Paravirtualization）：需要更改操作系统内核，同时需要 Guest OS 和 Hypervisor 兼容。
  • 内核半虚拟化（Kernel Paravirtualization）：需要更改操作系统内核，同时需要 Guest OS 和 Hypervisor 兼容。

操作系统虚拟化

在容器模型中，虚拟层是通过创建虚拟操作系统实例实现的，它再指向根操作系统的关键系统文件，返回指针驻留在操作系统容器叐保护的内存中，提供低内存开销，因此虚拟化实例的密度很大。密度是容器架构相对于Ⅰ型和Ⅱ型架构的关键优势之一，每个虚拟机都需要一个完整的客户机操作系统实例。

在容器内，用户可以使用特定的应用程序、补丁 Fixes（但不是 Service Pack，它会更 改操作系统 Kernel 的共享系统文件）和操作系统服务组件自定义客户机实例。对于那些在多数客户机容器中会使用到的服务或应用程序，它们所需要的功能应该安装到根操作系统中，在客户机实例中使用类似于模板的方法自动获得这些功能。

托管

Type1（裸金属） Hypervisor 架构包括主要的企业级虚拟化产品，这种类型的 Hypervisor 直接运行在物理服务器上，为虚拟机提供最好的性能。通过Intel和AMD CPU的虚拟化指令集，Type 1 Hypervisor 能够让虚拟机在某些情况下，性能接近或者达到物理机的性能。

Type 1 架构有三个分类

独立型（Stand-alone）

在独立型Hypervisor中，特别定制的Hypervisor提供所有硬件虚拟化和VMM功能。这种架构体现在VMware ESXi Server产品线中。

混合型（Hybird）

混合型架构包括一个软件模型，一个“痩”Hypervisor联合一个父分区提供硬件虚拟化，它提供了虚拟化监视功能。这类模型主要包括微软的 Hyper-v 和基于Xen的Hypervisor。

父分区也叫作 Dom0，它通常是一个运行在本地的完整的操作系统虚拟机，并具有根权限。

与 VMware 类似，所有的混合型产品都为客户机提供了半虚拟化驱劢，仍而提高网络和 I/O 性能，并实现半虚拟化驱动的客户机必须遍历父分区的 I/O 堆栈，因此客户机的性能会下降。

混用型

基于Linux的内核虚拟机（KVM） Hupervosor模型提供了一个独立无二的Type1型架构，它不是在裸机上执行Hypervisor ，KVM利用开源Linux（包括RHEL、SUSE 和 Ubuntu 等）作为基础操作系统，提供一个集成到内核的模块（叫作KVM）实现硬件虚拟化，KVM模块在用户模式下执行（与独立型和混合型 Hypervisor 不一样，它们都运行在内核/根模式下），但可以让虚拟机在内核级权限使用一个新的指令执行上下文（叫作客户机模式）。

KVM 使用一个经过修改的开源 QEMU 硬件仺真包提供完整的硬件虚拟化，这意味着客户机操作系统丌需要操作系统半虚拟化。不 VMware 类似，Linux KVM 充分利用 VirtIO 作为实现 I/O 半虚拟化的框架，它利用内置在内核 QEMU 中的用户模式 VirtIO 驱动来增强性能。

VDI架构介绍

用户访问层（user access layer）

用户访问时用户进入VDI的入口。用户通过支持VDI访问协议的各种设备，如电脑、痩客户端、上网本和手持移动设备等来访问。

虚拟架构服务层(virtual infrastructure service layer)

虚拟架构服务层为用户提供安全、规范和可用的桌面环境。用户访问层通过特定的显示协议和该层通信，如VMware 使用的时RDP和PCoIP，Cirtix使用HDX,redhat使用SPICE等。

虚拟架构服务层有如下主要组件和功能：

  • Hypervisor：为虚拟桌面的虚拟机提供虚拟化运行环境。这些虚拟机就叫做用户虚拟左面
  • 用户虚拟桌面（hosted virtual desktop)：虚拟机里面运行的桌面操作系统和应用就是一个用户虚拟桌面
  • 连接管理器(connetion broker)：用户的访问设备通过连接管理器来请求虚拟桌面。它管理访问授权，确保只有合法的用户能够访问VFI
  • 管理和提供服务：管理和提供服务集中化管理虚拟架构，它提供单一的控制界面来管理多项任务。它提供镜像管理、生命周期管理和监控虚拟桌面
  • 高可用性服务：高可用性（HA）服务保存虚拟虚拟机在关键的软件或者硬件出现故障时能够正常运行。HA可以是连接管理器功能的一个部分，为无状态HVD提供服务，也可以为全状态HVD提供单独的故障转移服务

HVD虚拟机分配模式：

  • 永久

    一个永久HVD分配给特定的用户。用户每次登录时，连接都是同一个虚拟机，用户在这个虚拟机安全或者修改的应用和数据将保存下来，用户注销后也不会丢失。这种独占模式非常适合需要自己安装更多的应用程序，将数据保存在本地，保留当前状态，一边下次登陆后可以继续工做作的情况。

  • 非永久

    一个非永久HVD是临时分配给用户的。当用户注销后，所有对镜像的变化都被丢弃。接着，这个桌面进入池中，可以被另外一个用户连接使用。用户的个性化桌面和数据将通过属性管理、目录重定向等保留下来。特定的应用程序将通过应用程序虚拟化技术提供给非永久HVD。

存储服务层（storage service layer）

存储服务层存储用户的个人数据、属性、母镜像和实际的虚拟桌面镜像。虚拟架构服务调用存储协议来访问数据。VDI里面常用到的存储协议有NFS（network file system），CIFS（common internet file system）、iSCSI和Fibre Channel等。

虚拟化原理

KVM原理

KVM架构

从虚拟机的基本架构上来区分，虚拟机一般分为两种，我们称之为类型一和类型二。

  • 类型一虚拟机时在系统加电之后首先加载运行虚拟机监控程序，而传统的操作系统则运行在其创建的虚拟机中。

  • 类型二虚拟机监控程序，在系统加点之后仍然运行一般意义上的操作系统，也就是俗称的宿主机操作系统。

KVM内核模块在运行时按需加载进入内核空间运行。KVM本省不执行任何设备模拟，需要用户空间程序QEMU通过/dev/kvm接口设置一个虚拟客户机的地址空间，向它提供模拟的I/O设备，并将它的时评显示映射会宿主机的显示屏。

KVM模块

KVM模块时KVM虚拟机的核心部分。其主要功能时初始化CPU硬件，打开虚拟化模式，然后将虚拟客户机运行在虚拟模式下，并对虚拟客户机的运行提供一定的支持。为了软件的简洁和性能，KVM仅支持硬件虚拟化。除了处理器的虚拟化，内存虚拟化也是由 KVM 模块实现的。

Qemu模型

QEMU本身并不是KVM的一部分，其自身就是一个著名的开源虚拟机软件。与KVM不同，QEMU虚拟机是一个纯软件的实现，所以性能低下。其优点时在支持QEMU本身编译运行的平台上就可以实现虚拟机的功能，甚至虚拟机可以与宿主机不是同一个架构。

Libvirt

QEMU-KVM 工具可以创建和管理 KVM 虚拟机，RedHat 为 KVM 开収了更多的辅助工具。libvirt是一套提供了多种语言接口的API，为各种虚拟化工具一共一套翻遍、可靠的编程接口，不仅支持KVM，而且支持Xen等其他虚拟机。Libvirt不仅提供了API，还自带一套基于本的管理虚拟的命令virsh，可以通过使用virsh命令来使用libvirt的全部功能。virt-manager时图形用户界面，它是一套用python编写的虚拟机管理图形界面，用户可以通过它直观的操作不同虚拟机。virt-manager 就是 利用 libvirt 的 API 实现的。Libvirt 是一个软件集合，便亍使用者管理虚拟机和其他虚拟化 功能，比如存储和网络接口管理等。Libvirt 概括起来包括一个 API 库、一个 daemon（libvirtd） 和一个命令行工具（virsh）。

Libvirt的主要功能如下：

  • 虚拟机管理：

    包括不同的领域生命周期操作，比如启动、停止、暂停、保存、恢复和迁移。

  • 远程机器支持：

支持多种网络远程传输，可以使用最简单的SSH，不需要额外配置工作。

  • 存储管理：

    任何运行了libvirt deamon的主机都可以用来管理不同类型的存储，可以创建不同格式的文件映像（qcow2、vmdk、raw等）、挂载NFS共享，列出现有的LVM卷组、创建新的LVM卷和逻辑卷。对未处理过的磁盘设备分区，挂接iSCSI共享等。因为libvirt可以远程工作，所有这些都可以通过远程主机使用。

  • 网络接口管理：

    任何运行了libvirt deamon的主机都可以用来管理物理和逻辑的网络接口。可以列出现有的接口卡，配置、创建接口，以及桥接、vlan和关联设备等，通过netcf都可以支持。Libvirt 虚拟网绚使用防火墙觃则作为路由器，让虚拟机可以透明访问主机的网绚。

使用KVM管理工具

virsh（virtual shell）是由一个名为 libvirt 的软件提供的管理工具，提供管理虚拟机更高级的能力。virsh 大部分的功能与XM 一样，你可以利用 virsh 来启动、删除、控制和监控 KVM 中所有的虚拟机。
